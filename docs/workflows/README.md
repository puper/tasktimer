# 开发流程

本目录描述 TaskTimer 的开发工作流程。

## 工作流阶段

### 1. 研究阶段 (Research)

研究阶段产出存放于 `.artifacts/research/` 目录：
- 代码结构分析
- 模式识别
- 依赖关系映射

### 2. 计划阶段 (Planning)

计划阶段产出存放于 `.artifacts/plan/` 目录：
- 实现计划
- 任务分解
- 风险评估

### 3. 执行阶段 (Execution)

执行阶段产出存放于 `.artifacts/execute/` 目录：
- 执行日志
- 变更记录
- 进度跟踪

### 4. QA 阶段 (Quality Assurance)

QA 阶段产出存放于 `docs/qa/` 目录：
- 测试报告
- 代码审查
- 验证清单

## 变更流程

1. 创建研究文档，理解现有代码
2. 制定实现计划，明确任务边界
3. 按计划执行，记录关键决策
4. 执行 QA 验证，确保质量

## 验证要求

每次变更需满足：
- 所有测试通过
- 代码格式化
- 示例可运行
- 文档同步更新
