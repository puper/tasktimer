# 约定规范

本目录定义 TaskTimer 项目的代码和文档规范。

## 语言约定

- **默认语言**: 中文
- **代码注释**: 中文
- **文档编写**: 中文
- **提交信息**: 中文或英文均可

## 代码规范

### 命名约定

- 导出类型使用大写字母开头：`Engine`, `Task`, `Tx`
- 私有字段使用小写字母开头：`slots`, `jobToSlot`
- 方法名使用动词：`Add`, `Get`, `Delete`, `Start`, `Stop`

### 结构约定

- 每个公共类型和方法应有注释
- 复杂逻辑应有行内注释说明
- 使用 `gofmt` 格式化代码

### 并发约定

- 所有公共操作通过 `Execute()` 事务方法
- 避免在回调中执行阻塞操作
- 回调必须处理 panic

## 文档规范

### 文档结构

- 使用 Markdown 格式
- 每个目录包含 README.md 作为索引
- 保持文档简洁，指向代码而非重复描述

### 更新要求

- 新增功能需更新相关文档
- API 变更需更新 AGENTS.md
- 设计决策需记录在 docs/decisions/

## 测试规范

- 测试函数命名：`Test<功能描述>`
- 使用 table-driven 测试处理多场景
- 并发测试使用 `sync/atomic` 验证结果
